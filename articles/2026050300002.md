## はじめに

最近のBeReal関連の情報露出事案を見ていて、ずっと頭に浮かんでいたのが『呪術廻戦』のあるシーンでした。

壊相の術式に対して、釘崎野薔薇が言う、

> 「私との相性 最悪だよ」

の場面です。

BeRealって、まさにこれなんですよね。

プライベート空間では強力な“良い術式”なのに、業務空間へ持ち込んだ瞬間、環境との相性が最悪になる。

しかも厄介なのは、術式そのものは別に悪ではないことです。

---

# そもそもBeRealはかなり優秀なSNS設計

BeRealは、

* 盛らない
* 加工しない
* 今を共有する
* 雑な日常を見せる

という方向へ全振りしています。

これは、従来SNSの

* 映え疲れ
* 演出疲れ
* 承認疲れ

に対するカウンターとしてかなり優秀。

つまりプライベート空間では、

> 「ちゃんとしてない状態」

そのものが価値になる。

だから、

* 深夜コンビニ
* 作業机
* 散らかった部屋
* 適当な昼飯

みたいなものがコンテンツになる。

Instagramが“作品展示”なら、BeRealは“雑談”です。

---

# ただし業務空間との相性が悪すぎる

問題は、BeRealのルールが業務空間と根本的に噛み合わないことです。

かなり面白いぐらい真逆。

| BeReal世界での価値 | 業務空間での意味    |
| ------------ | ----------- |
| 即撮影がリアル      | 確認不足        |
| ノーガード感が面白い   | 情報露出        |
| 背景込みの日常共有    | 映り込み事故      |
| 加工しない誠実さ     | マスキング不足     |
| “今この瞬間”が価値   | 熟慮不足        |
| 友達限定の安心感     | 油断          |
| 雑さが親密さになる    | 雑さが脆弱性になる   |
| 日常感がコンテンツ    | オフィス情報が漏洩対象 |

プライベートでは“魅力”だったものが、業務ではそのまま“リスク”へ反転する。

この反転がかなり綺麗です。

---

# BeRealは「考える前に動かせる」

特に強いのが、

* ランダム通知
* 時間制限
* 同時撮影

の組み合わせ。

普通のSNSなら、

1. 撮る
2. 見返す
3. 加工する
4. 投稿を考える

という“熟慮フェーズ”があります。

でもBeRealは、

> 「今撮る」

がゲームルール。

つまりユーザーは、

* 背景確認
* PC確認
* ホワイトボード確認

より、

> 「間に合うか」

を優先しやすい。

これ、かなり“反射行動特化型UX”なんですよね。

---

# しかも「閉じた空間」に見える

さらにBeRealは、Xほど公開SNS感がない。

感覚的には、

> 「友達グループの内輪ノリ」

に近い。

すると人間は一気に警戒を解きます。

* 少しぐらい大丈夫
* 身内だけ
* この場限り

と思いやすい。

ただ実際には、

* スクショ可能
* 保存可能
* 転送可能
* 再共有可能

なので、技術的には普通に“永続化されたデータ”。

ここにかなり大きな認知ギャップがあります。

---

# 「雑に振る舞うほど正しい」という術式

冷静に考えると、BeRealってかなり特殊です。

普通、情報管理では、

* 慎重
* 整理
* 確認
* 秘匿

が求められる。

でもBeRealは逆方向。

むしろ、

> 「雑に振る舞うほどリアルで良い」

という価値観を持っている。

つまりBeRealの術式は、

> “確認不足を文化として肯定する”

方向に働く。

プライベートではこれが楽しい。

でもオフィスでは、完全に相性が悪い。

まさに、

> 「私との相性 最悪だよ」

状態。

---

# 情シス視点で見ると「教育不足」だけでは弱い

この手の話は、

> 「SNSリテラシー教育を徹底しましょう」

で終わりがちです。

もちろん教育は必要です。

ただ、BeRealって、

* 即時性
* 反射性
* 内輪感
* ノーガード文化

をかなり強く設計へ組み込んでいる。

つまり、

> 「普通の人が、普通に事故る」

方向へ自然に誘導される。

だから重要なのは、

> 「絶対ミスするな」

ではなく、

> 「ミスしても致命傷にならない」

設計。

たとえば、

* ホワイトボードへ個人情報を書かない
* 撮影前提の空間設計
* 背景映り込みを考慮する
* 私物スマホルールを整理する

など。

ゼロトラストって、システムだけではなく、
人間の認知や行動設計まで含めて考える時代なんだな、と感じます。
