# IT民俗学：いつからOS標準のマルウェア対策へ身を委ねるようになったのか

昔、パソコンを買ったら最初にやることがありました。

* ウイルスバスターを入れる
* Nortonを入れる
* McAfeeを入れる

そして、こう言われていました。

> Windows標準のセキュリティ機能なんて信用できない。

それが常識だったように思います。

ところが今、新しいWindows PCを買っても、私は何もインストールしません。

Windows Defenderが動いている。

それで十分だと思っています。

昔は「あんなものでは守れない」と言われていたのに、私たちはいつからOS標準のマルウェア対策へ身を委ねるようになったのでしょうか。

## Windowsだけでは守れなかった時代

今では少し想像しづらいのですが、Windows XP初期には、

* 標準ファイアウォールは無効
* ウイルス対策機能なし
* スパイウェア対策なし
* 振る舞い検知なし

という状況でした。

一方、サードパーティ製品は、

* リアルタイムスキャン
* メール検査
* HTTP通信検査
* スパイウェア対策
* パーソナルファイアウォール

などを次々と実装していきます。

さらに更新頻度も違いました。

Windows Updateが月単位、あるいはサービスパック単位で更新されていた時代に、アンチウイルスベンダーは、

```text
新しいウイルス発見
↓
数時間後に定義ファイル配信
```

という世界を作っていました。

当時の利用者は、

> 新しい脅威へいち早く追従してくれること

そのものを買っていたのです。

つまり、

```text
Windows ＋ アンチウイルスソフト ＝ 完成品
```

でした。

## 第一世代：ファイルを守る時代

2000年のILOVEYOUワームは、世界中へ爆発的に広がりました。

参考：

* FBI Archive: Love Bug Virus Warning
  https://archives.fbi.gov/archives/news/stories/2000/may/lovebug051000
* CERT Advisory CA-2000-04 Love Letter Worm
  https://www.cert.org/historical/advisories/CA-2000-04.cfm

この時代、悪いものは「ファイル」としてやってきました。

```text
悪いもの = ファイル
```

だから、

```text
ファイル
↓
シグネチャ照合
↓
検知
```

という考え方が成立していました。

## 第二世代：OSを守る時代

2003年のBlaster。

2004年のSasser。

Windows RPCの脆弱性を悪用し、ネットワークへ接続しただけで感染しました。

参考：

* Microsoft Security Bulletin MS03-026
  https://learn.microsoft.com/en-us/security-updates/securitybulletins/2003/ms03-026
* CISA Alert: Blaster Worm
  https://www.cisa.gov/news-events/alerts/2003/08/13/blaster-worm

メールを開いていない。

怪しいサイトも見ていない。

それなのに感染する。

「何もしてないのに壊れた」が、本当に起きていた時代です。

ここで守るべきものが変わりました。

```text
ファイル
↓
OSそのもの
```

Windows XP SP2で標準ファイアウォールが有効化されたのも、この頃でした。

## 第三世代：挙動を守る時代

2009年のOperation Aurora。

参考：

* Google Official Blog: A new approach to China
  https://googleblog.blogspot.com/2010/01/new-approach-to-china.html

正しいPDF。

正しいWebサイト。

正しいファイル。

それらが悪用され始めます。

すると、

```text
このファイルは悪い
```

ではなく、

```text
この挙動は怪しい
```

を見なければならなくなりました。

ヒューリスティック。

サンドボックス。

振る舞い検知。

防御の重心が少しずつ移っていきます。

## 第四世代：侵害を前提とする時代

2017年のWannaCry。

そしてEmotet。

参考：

* Microsoft Security Blog: Customer Guidance for WannaCrypt attacks
  https://blogs.microsoft.com/blog/2017/05/12/customer-guidance-wannacrypt-attacks/
* CISA Alert AA20-280A: Emotet Malware
  https://www.cisa.gov/news-events/cybersecurity-advisories/aa20-280a

ここで防御側は、一つの現実を受け入れ始めます。

> 侵入を100％防ぐことはできない。

すると、

```text
感染させない
```

から、

```text
侵害された後に何をしたか
```

を見る時代になります。

- EDR
- XDR
- SOC

しかし、ここでもう一つの変化が起きています。

## 大事なものはPCの中になくなった

昔、PCは**金庫**でした。

- メール
- 文書
- 業務データ

大事なものはPCの中にありました。

だから、

```text
PCを守る ＝ 資産を守る
```

だったのです。

ところが今は違います。

* Microsoft 365
* Google Workspace
* Salesforce
* SharePoint
* Box

本当に大事なものは、PCの外へ移りました。

```text
PC
↓
クラウド
↓
データ
```

という構造です。

すると攻撃者の目的も変わります。

2021年のColonial Pipeline事件では、VPNアカウントの侵害が社会インフラ停止へ発展しました。

2023年のMGM Resorts事件では、ヘルプデスクへの電話一本から組織全体が麻痺しました。

参考：

* CISA: DarkSide Ransomware Best Practices
  https://www.cisa.gov/news-events/cybersecurity-advisories/aa21-131a
* CISA/FBI Advisory on Scattered Spider
  https://www.cisa.gov/news-events/cybersecurity-advisories/aa23-320a

どちらの事件でも、攻撃者が本当に欲しかったのはPCそのものではありません。

欲しかったのは、

```text
クラウドへ入るための資格情報
```

でした。

## PCは金庫ではなく、鍵束になった

今のPCには、

* パスワード
* セッションCookie
* MFAトークン
* アクセストークン

が存在します。

つまり、

```text
PC = 鍵束
```

になりました。

本当の資産は、その先にあります。

だから最近の攻撃は、「**PCを壊す**」のではなく、「**Identityを奪う**」方向へ進化しています。

## 境界型防御は終わった

昔は、

```text
社内 = 安全
社外 = 危険
```

でした。

しかし、

* SaaS
* モバイル
* テレワーク

によって、

```text
大事なもの
↓
どこからでもアクセスできる
```

ようになりました。

最もわかりやすい防壁、物理的な境界が消えたのです。

だから、「**社内だから信用する**」という考え方は成立しなくなりました。

そこで生まれたのが、

```text
誰も信頼しない
毎回確認する
```

というゼロトラストの考え方です。

## なぜ「OS標準のマルウェア対策で十分」と感じるようになったのか

ここで、最初の問いに戻ります。

私たちはなぜ、OS標準のマルウェア対策で十分だと感じるようになったのでしょうか。

おそらく理由は二つあります。

一つは、Windows標準の防御機能が成熟したことです。

- Defender
- SmartScreen
- Secure Boot
- Credential Guard

これらによって、少なくとも個人が利用する一台のPCを守るという観点では、「**Windows標準機能で十分**」と感じられるようになりました。

もう一つは、大事なものがPCの中だけに収まらなくなったことです。

個人利用であれば、守りたい範囲は比較的見えやすい。

自分のPC。

自分のブラウザ。

自分のアカウント。

自分が接続するサービス。

この範囲であれば、OS標準の防御機能と各サービス側の認証・確認によって、ある程度の安心感を得られます。

しかし企業では事情が違います。

企業にとってPCは、もはや単体で完結する道具ではありません。

一台一台のPCは、

* Identity
* SaaS
* ファイル共有
* チャット
* 業務システム

へ接続するための入口です。

つまり、企業におけるPCは、「**一台の端末**」ではなく、「**組織の資産へ至る扉**」なのです。

個人の視点では、自分のPCが守られていればよい。

しかし管理者の視点では、攻撃対象はPCの数だけ存在します。

一つの端末が侵害される。

資格情報が奪われる。

クラウドへアクセスされる。

横展開される。

すると、一台のPCの問題は組織全体の問題になります。

だから企業では、OS標準のマルウェア対策だけでは足りません。

必要になるのは、

```text
このPCは守られているか
```

だけではなく、

```text
このPCで起きたことが、組織全体へどう波及するか
```

を観測する仕組みです。

そこでEDR、XDR、SOCといった考え方が意味を持ちます。

個人にとってのPCは、今も「自分の道具」です。

しかし企業にとってのPCは、組織の境界上にあるエンドポイントです。

だから、

```text
個人PC → 標準機能で守る

企業PC → 組織全体の一部として観測する
```

という違いが生まれる。

「OS標準のマルウェア対策で十分」と感じるようになったのは、Windowsが成熟したからだけではありません。

PC単体を守ることと、組織全体を守ることが、別の問題として分かれてきたからなのかもしれません。


## では、AI時代はどうなるのだろう

ここまでの話だけなら、

> 組織にとって、PCは資産の扉を開く鍵束になった。

で終われます。

でも最近、少し違う変化を感じています。

- Claude Code
- Codex
- Cursor
- Copilot Agent

彼らは、

* リポジトリ
* README
* .env
* SSH鍵
* ターミナル履歴

を読みながら、ローカルPCに情報を持ち、人に代わって仕事をします。

一見すると、かつて手放した資産がPCへ戻ってきたようにも見えます。

しかし実際には少し違うのかもしれません。

戻ってきたのは資産そのものではありません。

```text
何が重要で、
どこにあり、
どう辿り着くのか
```

というコンテキストです。

## AIは鍵束を持った秘書になる

AIエージェントは、

* GitHubへアクセスする
* AWS CLIを実行する
* kubectlを叩く
* Pull Requestを作成する

つまり、

```text
AI = 権限を委譲された主体
```

になり始めています。

もし攻撃者がそれを騙したら。

もしプロンプトインジェクションによって行動を書き換えられたら。

AIは、信頼できる相棒から「攻撃者のスパイ」に成り果てるかもしれません。

ここで、頭をよぎる二つの作品があります。

『攻殻機動隊 STAND ALONE COMPLEX』の笑い男事件。

あの作品が描いていたのは、

> 自分が見ている情報を、どこまで信用してよいのか

という問いでした。

そして『PSYCHO-PASS』のシビュラシステム。

あの作品が描いていたのは、

> 判断そのものを、どこまで委ねてよいのか

という問いだったように思います。

AIエージェントの時代、私たちはその二つを同時に突き付けられているのかもしれません。

エージェントが見ている情報は正しいのか。

エージェントの判断を信じてよいのか。

そして、もしそのエージェントが騙されたなら。

## 私たちは、その時代で最も危険な信頼を守っている

振り返ってみると、

```text
ファイルを信用しない
↓
OSを信用しない
↓
侵害後を前提にする
↓
Identityを守る（イマココ）
```

と、守るべきものは変わり続けてきました。

その背景には、

```text
大事なものはどこにあるのか
```

という変化があります。

昔、PCは金庫でした。

次に、PCは鍵束になりました。

そして今、私たちはその鍵束をAIへ預け始めています。

もしかすると次の時代に守るべきものはファイルでも、PCでも、Identityでもなく、

> 人ではない協力者へ、どこまで判断と権限を委ねてよいのか

という信頼そのものなのかもしれません。
